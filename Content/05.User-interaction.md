# User Interaction

 User interaction allows a script to receive input directly from the terminal. In Bash, we use the `read` command to capture input from the user.


## `read` Basic Input
 The simplest way to get input from a user:
 
```bash
#!/bin/bash
 
echo "What is your name?"
read name
echo "Hello, ${name}!"
```

>[!important]
>`read name` waits for user input, and the entered value is stored inside the variable name.
 
 
## `read -p`  Inline Prompt
Instead of using `echo`, you can display the prompt directly with `-p`.
```bash
#!/bin/bash
 
read -p "Enter your name: " name
read -p "Enter your city: " city
 
echo "Hello ${name} from ${city}!"
```
 
## `read -s` Hidden Input (e.g. passwords)
The `-s` option hides the user's input while typing.
```bash
#!/bin/bash
 
read -p "Username: " username
read -s -p "Password: " password
echo ""   # move to a new line after hidden input
 
echo "Logging in as ${username}..."
```
 
Output
```
Username: Ahmad
Password:
Logging in as Ahmad...
```
 
>[!Important]
>The password is not shown while typing because of `-s`.
