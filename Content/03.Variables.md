# Variables

Variables are used to store data in a Bash script. They allow you to reuse values instead of repeating the same information multiple times.

Using variables makes your scripts cleaner, more flexible, and easier to maintain.


## Declaring and Using Variables

Let’s begin with a simple example to understand how variables work in Bash.

```bash
#!/bin/bash
 
name="Ahmad"
age=24
language="Bash"
 
echo "Name    : ${name}"
echo "Age     : ${age}"
echo "Language: ${language}"
```
Output
```
Name    : Ahmad
Age     : 24
Language: Bash
```
### How variables work
1. Declaration
```bash
name="Ahmad"
age=24
```
- No spaces around `=`
- Strings should be inside quotes
- Numbers do not require quotes
- Variable names are case-sensitive (Name ≠ name)

2. Accessing a Variable
- To use a variable, prefix it with `$`:
```bash
${name}
```
- You will commonly see variables written like this:

```bash
"${name}"
```
Quoting prevents problems if the value contains spaces.

You can see this here:
```bash
name="Ahmad Ali"
echo "${name}"
```
Without quotes, Bash may treat Ahmad and Ali as separate words.
 
## Global vs Local Variables
 
By default, variables in Bash are global that means once you create them, they are accessible anywhere in the script.

 
```bash
#!/bin/bash
 
language="Bash"   # global — accessible everywhere
 
show_info() {
    local version="5.1"   # local — only exists inside this function
    echo "Language: ${language}"
    echo "Version : ${version}"
}
 
show_info
 
echo "Version outside function: ${version}"   # prints nothing
```
 
Output
```
Language: Bash
Version : 5.1
Version outside function:
```

### What Happened?

- `language` is **global** → accessible everywhere.
- `version` is **local** → exists only inside `show_info()`.
- Outside the function, `version` does not exist.

### Why Local Variables Matter

Using `local` inside functions:

- Prevents accidental modification of global variables  
- Makes functions safer and easier to reuse  
- Reduces bugs in larger scripts
