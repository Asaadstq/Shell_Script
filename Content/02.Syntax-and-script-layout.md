# Syntax and Script Layout

Bash scripts follow a clear layout, and keeping your script organized makes it easier to read, debug, and maintain. 

Let’s examine a simple example script named `greet.sh`.
## Script Structure
- Script name `greet.sh`
```bash
#!/bin/bash
 
# -----------------------------------------------
# Script: greet.sh
# Purpose: Show basic script structure
# -----------------------------------------------
 
# Variables at the top
name="Abdullah"
 
# Logic in the middle
if [[ -n "${name}" ]]; then
    echo "Hello, ${name}!"
fi
```
### 1. Shebang Line
 ```bash
 #!/bin/bash
```
- This must be the first line it tells the system to use the Bash interpreter to run the script.

### 2. Comments Section
- Comments start with `#` they explain what the script does, and it's ignored when the script runs.

### 3.  Variables Section
- `name="Alice"`
   - variables store data.
   - No spaces around `=`.
   - Strings should usually be placed inside quotes.
### 4. Logic Section
- This is where the script performs actions.
